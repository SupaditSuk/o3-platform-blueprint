# 11 Subscription, CRM, and Governance

## Package Structure

Recommended packages:

1. Basic
2. Starter
3. Professional
4. Business
5. Enterprise

## Package Logic

Basic:

- Free account
- Limited assessment
- Sample dashboard
- Limited AI credits
- Basic Academy access

Starter:

- Upload template
- Basic dashboard
- Limited AI Advisor
- Basic reports

Professional:

- Full dashboard MVP
- More AI tools
- Export reports
- More credits

Business:

- Multi-user
- Advanced dashboard
- Team permissions
- More AI credits
- CRM and action tracking

Enterprise:

- Custom integration
- SSO
- Advanced security
- Custom dashboard
- SLA
- Dedicated support

## Entitlement System

Every feature should be controlled by entitlement, not hardcoded.

Entitlement examples:

- Can upload data
- Can access turnover dashboard
- Can use AI Advisor
- Can export report
- Can access Survey Studio
- Can access advanced benchmark
- Monthly AI credit limit

## Usage Tracking

Track product usage from day one.

Events:

- User registered
- Company created
- Assessment started
- Assessment completed
- Template downloaded
- File uploaded
- Validation failed
- Data imported
- Dashboard viewed
- AI prompt sent
- AI tool used
- Course started
- Course completed
- Subscription upgraded

## CRM Data

CRM should combine:

- Lead source
- Company profile
- Product usage
- AI usage
- Academy progress
- Subscription stage
- Sales notes
- Customer success status

## Governance

O³ should maintain governance over:

- Data access
- AI usage
- Feature entitlement
- Template version
- Product visibility
- Benchmark eligibility
- Audit history

## Admin Portal

Admin should be able to:

- Manage users
- Manage companies
- Manage subscriptions
- Configure packages
- Configure AI credits
- View usage analytics
- Manage Academy courses
- Manage template versions
- Review AI logs
