# Book 05: Information Architecture

## Purpose

This book defines how data flows through the O³ platform, including data models, data flow patterns, data governance, and information lifecycle management.

## Chapters

1. Data Flow Architecture
2. Data Classification and Sensitivity
3. Data Lifecycle Management
4. Data Governance Framework
5. Data Integration Patterns
6. Reporting and Analytics Data Model

## Key Takeaways

- Data flows: Upload → Validate → Import OWDS → Insight Engine → Dashboard/AI
- Data classified by sensitivity (public, internal, confidential, restricted)
- Data lifecycle: create, store, use, archive, delete
- OWDS is the canonical data model for all workforce data
- Benchmark data is stored separately with anonymization
